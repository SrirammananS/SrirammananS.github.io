UNIT 1: Networking Models & Basics

OSI Model (Open Systems Interconnection)

Layer Name Function
7 Application User interface, network services
6 Presentation Data translation, encryption
5 Session Session management
4 Transport End-to-end connections (TCP/UDP)
3 Network Routing, IP addressing
2 Data Link MAC, switching, framing
1 Physical Cables, signals, bits

TCP/IP Model

Layer Protocols Functions
Application HTTP, FTP, DNS, SMTP Application services
Transport TCP, UDP Communication between hosts
Internet IP, ICMP, ARP Logical addressing, routing
Network Access Ethernet, Frame Relay Physical addressing and access

MAC Address

  • Format: 00:1A:2B:3C:4D:5E (48-bit)
  • OUI: First 24 bits (Organisational Unique Identifier)

IP Versions

Version Length Format Example
IPv4 32-bit 192.168.1.1
IPv6 128-bit 2001:0db8:85a3::8a2e:0370:7334

IP Header Flags

Flag Meaning
DF Don’t Fragment
MF More Fragments

TCP Header Flags

Flag Description
SYN Synchronize
ACK Acknowledgment
FIN Finish
RST Reset
PSH Push
URG Urgent

TCP 3-Way Handshake

  1. SYN → (Client to Server)
  2. SYN-ACK ← (Server to Client)
  3. ACK → (Client to Server)

Encapsulation vs De-encapsulation

  • Encapsulation: App → Trans → Net → Link → Physical
  • De-encapsulation: Physical → Link → Net → Trans → App

UNIT 4: VLANs & Layer 2 Concepts

VLAN Types

VLAN Type Description
Access Assigned to single VLAN
Trunk Carries multiple VLANs
Dynamic Uses VTP to propagate VLAN info

Switchport Modes

Mode Function
Access Single VLAN
Trunk Multiple VLANs
Dynamic Desirable Actively tries to form trunk
Dynamic Auto Passively waits to form trunk

VTP Modes

Mode Description
Server Create/Modify VLANs
Client Receives VLAN info only
Transparent Doesn’t participate, forwards only
